Inception

A Docker-based infrastructure project — by dopereir — 42 School

About

Inception is a system administration project from 42 School. Each service runs in its own Docker container, built from a custom Dockerfile based on Alpine Linux — no pre-built images are used. Containers communicate over a private Docker network, isolated from the host. Persistent data is stored in named volumes shared between services, ensuring data survives container restarts. The whole infrastructure is orchestrated with Docker Compose and runs inside a virtual machine.

Mandatory services

NGINX Reverse proxy with TLSv1.3 only, entry point for all traffic
WordPress + PHP-FPM CMS running without a web server, served via FastCGI
MariaDB Relational database for WordPress data storage

Bonus services

Redis Object cache for WordPress to reduce database load
vsftpd FTP server with access to the WordPress volume
Static site This page — a simple HTML/CSS site served by NGINX
Adminer Web-based database management interface for MariaDB
